# 动态漫剧创作工具实施计划

> **面向执行型 Agent：** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development`（推荐）或 `superpowers:executing-plans` 按任务逐条执行。步骤使用 checkbox（`- [ ]`）语法跟踪。

**Goal:** 搭建一个面向小说/短篇作者的 Web 创作台 MVP，完成“单章节文本输入 -> 结构化剧本稿 -> 动态漫剧草稿 -> 中度编辑 -> 站内预览 -> 导出 MP4”的闭环。

**Architecture:** 采用前后端分离但目录统一管理的方案。前端负责项目创建、剧本确认、分镜编辑和导出交互；后端负责项目管理、脚本拆解、任务编排、资产落盘和视频导出，AI 生成能力先通过可替换的 provider 适配层接入。

**Tech Stack:** Vite + React + TypeScript + Tailwind CSS、FastAPI + Pydantic、PostgreSQL、Celery + Redis、本地文件存储、FFmpeg、第三方 LLM / 文生图 / TTS API

## 实施原则

- 第一阶段只做单章节，不做多章节连续项目
- 第一阶段先支持本地文件存储，不先上对象存储
- 第一阶段先支持“脚本确认后生成草稿”，不做完全黑盒一键视频
- 第一阶段优先保证可恢复、可重试，不追求最强角色一致性
- 所有生成链路先支持 provider 抽象，但只接 1 组默认实现
